System Cards

System cardss are virtual cards designed to quickly access the contents of the System Book. Cards primarily contain core-based policies and secondarily information about the implementation of those policies. Other policies linked to a particular policy are accessible on the System Board.

As a feature of system cards, visual charts are designed to gain quick insight for each process found in the System Book. It should be sufficient to follow the information on the visual cards to perform a certain activity. Multiple cards can be used together for complex transactions.

  • Draft System Card
  • Sample System Card

General Information About System Card

Each system card has a specific number and name. The number identifies the theme and theme sub-title it relates to the system. The name of the system board refers to the topic to which it relates

System boards are prepared as two-sided. On the front of the card with all mandatory fields,

  • Definition
  • Policies
  • Processes
  • Prosedures

is found. On the back side with non-mandatory fields,

  • Workflow Charts
  • Support Documentation

takes place.
